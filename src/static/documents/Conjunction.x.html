Conjunction

The well-known rules for conjunction in propositional logic are the following: $$\infer{\land Intro}{P\land Q}{P&Q}$$ $$\infer{\land Elim} {P} {P\land Q}$$ $$\infer{\land Elim} {Q} {P\land Q}$$

In the programming language LEAN, the rules function similarly. For example, in the LEAN syntax you write: 

And.right h

This extracts from the hypothesis h: $P \land Q$ the right conjunct P. Similarly, to get the left conjunct, you can use the following syntax in LEAN 

And.left h

We have to be specific in LEAN, indicating exactly which conjunct we want to extract. For the introduction of a conjunction of P and Q we write: 

And.intro P Q


And.intro h1 h2

The latter formulation works for hypothesis h1, h2 respectively.

Next let us consider an example natural deduction proof involving both the introduction and the elimation rules for conjunction:

Example 1: $Q \land P \vdash P \land Q$ $$\infer{\land Intro} {P\land Q} {\infer{\land Elim} {P} {Q \land P} & \infer{\land Elim}{Q} {Q \land P}} $$

This proof is also done in LEAN; as you can see in the code editor on the page. This should give you an example to then continue the exercises in LEAN using the syntax for conjunction we just introduced. There are easier and harder exercises. You just remove the "sorry" placeholer and write your proof. You should be able to see LEAN messages regarding your proof.

Lastly, there is one helpful tool. When you write down natural deduction proofs on paper, you might want to use intermediary steps, similarly online. See for example the last hard exercise in the LEAN editor. It could be helpful to prove R as an intermediary goal there. LEAN allows you to do this by writing 

  show R from And.right (And.right h)
