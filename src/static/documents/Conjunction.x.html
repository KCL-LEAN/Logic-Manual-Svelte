Conjunction

The following pages discuss natural deduction for propositional logic in LEAN. We start with the rules for conjunction: $$\infer{\land Elim} {P} {P\land Q}$$ $$\infer{\land Elim} {Q} {P\land Q}$$ $$\infer{\land Intro}{P\land Q}{P&Q}$$

In the programming language LEAN, the rules function similarly. For example, to elimiate the conjunction and retain the right conjunct in LEAN you write: 

And.right h

From the hypothesis h: $P \land Q$ the above code extracts the right conjunct P. Similarly, to get the left conjunct, $Q$, you can use the following syntax in LEAN 

And.left h

We have to indicate exactly which conjunct we want to extract in LEAN. For the introduction of a conjunction of P and Q we write: 

And.intro P Q


And.intro h1 h2

The latter formulation works for hypothesis h1: $P$, h2: $Q$.

Next let us consider an example natural deduction proof involving both the introduction and the elimation rules for conjunction:

Example 1: $P \land Q \vdash Q \land P$ $$\infer{\land Intro} {Q\land P} {\infer{\land Elim} {Q} {P \land Q} & \infer{\land Elim}{P} {P \land Q}} $$

This proof is also done in LEAN; as you can see in the code editor. This should give you the model to continue the exercises in LEAN using the syntax for conjunction we just introduced. There are easier and harder exercises. You just remove the "sorry" placeholer and write your proof. LEAN will display error messages for your proof until you have provided a correct argument. Be careful with capitalization in the code.

Lastly, there is one helpful tool. When you write down natural deduction proofs on paper, you might want to use intermediary steps, similarly online. See for example the last hard exercise in the LEAN editor. It could be helpful to prove R as an intermediary goal there. LEAN allows you to do this by writing 

  show R from And.right (And.right h)
