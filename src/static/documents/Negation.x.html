Negation

The rules for negation in propositional logic are the following: $$\infer{\neg Intro} {\neg P} { {\underset{Q}{\overset{[P]}{\vdots}} & {\underset{\neg Q}{\overset{[P]}{\vdots}}}}}$$ $$\infer{\neg Elim } { P} {\underset{Q}{\overset{[\neg P]}{\vdots}} & {\underset{\neg Q}{\overset{[\neg P]}{\vdots}}}}$$

This works differently in LEAN. Negation $\neg P$ is here defined as $P \rightarrow \bot$, where $\bot$ is a contradiction such as $ 0\neq 0 $ for example. There are two main tactics for negation in LEAN. The first one is similar to negation introduction. If you want to prove $\neg P$ then you assume $P$ and show that it leads to a contradiction. This assumption is constructed similarly to the conditional and disjunctive rules using fun. Under some assumption you proof a contradiction (a False in LEAN). Then you get the desired result $\neg P$. It is written as follows:

fun h: P +=> show False from $\neg Q \quad Q$

Note that the order matters here, to show False from $Q$ and $\neg Q$, you must state the negated statement first.

The second tactic involving negation used in LEAN is the so called Ex falso quodlibet principle that anything can be derived from a contradiction. In LEAN this is done by the tactic: absurd. Suppose you are given some contradictory assumptions such as h1: $ P$ and h2: $\neg P$. Then you can prove any claim $Q$ simply by stating 

absurd h2 h1

Again here the order matters, the negated statement must come first.
