Introduction to LEAN: Syntax

We start by learning the syntax of propositional logic in LEAN. The syntax can be understood as the grammar of the language.

We denote sentence letters in the language of propositional logic by P,Q,R. All sentence letters are sentences and if $\phi, \psi$ are sentences then $\neg \phi, \phi \land \psi, \phi \lor \psi, \phi \rightarrow \psi, \phi \leftrightarrow \psi$ are also sentences in the language of propositional logic. In LEAN, sentence letters are of the type proposition or Prop. A proposition is a sentence letter which has a truth value.

LEAN editor

On the right side of the screen you can see the LEAN editor. It is split into two. In the upper half you can see code already and there you code yourself. In the lower half you see two displays: "mathlib-demo.lean:X.X" and "All Messages(X)". "mathlib-demo.lean:X.X" evaluates and verifies the code you click on. If you click in the upper half of the editor for example on line $3$ $(\#\text{check }P\lor Q)$, you can see that under "mathlib-demo.lean:X.X" two messages appear. If you open "Expected type" it will say P,Q are Prop (propositions), so LEAN expects to output a Prop. Under "Messages", you can see $P\lor Q$ is a Prop, as expected. The section "mathlib-demo.lean:X.X" displays the messages for the code that you work on, whereas if you open "All Messages", it will show messages for all the code written in the editor. #check evaluates the type of the expression. Try writing some more complex expressions using $P$ and $Q$!

Brackets

The Logic Manual defines conventions concerning brackets for compound sentences in section $2.3$. LEAN has its own conventions. It evaluates sentences from left to right and will assume brackets from left to right if you do not specify them. For example, if you input $P \land \neg P \lor Q$, LEAN will evaluate $(P \land \neg P) \lor Q$ automatically. If you want to evaluate $P \land (\neg P \lor Q)$ you have to specify the brackets.
