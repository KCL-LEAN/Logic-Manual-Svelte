Semantics of Propositional Logic

This section will discuss the semantics of Propositional Logic, following Chapter 5 of the Logic Manual. Please do not consider the first 2 lines of the code editor, this defines the interpret function in LEAN.

Semantic structures assign truth values to sentences in the language. We construct here a toy model and declare sentences A, B as false and C, D as true. Now following the definition of satisfaction (Def 5.2) you can evaluate the truth value of compound sentences. Instead of truth tables, LEAN automatically completes this interpretation. If you click on the #eval statements, you can see 'true' or 'false'.

You have to be precise with your brackets in LEAN. First, because we evaluate the interpret function, we need to put brackets around the sentences. If you use multiple connectives and don't specify the brackets, LEAN automatically adds brackets from left to right. The first example shows this: if you input $A \land \neg A \lor C$, LEAN will evaluate $(A \land \neg A) \lor C$.
