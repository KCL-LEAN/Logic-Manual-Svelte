Semantics of Propositional Logic

This section will discuss the semantics of propositional logic.

Semantic structures assign truth values to sentences in the language. We construct here a toy model and declare sentences P, Q as false and R as true. You can see this in the code editor. Now following the definition of satisfaction you can evaluate the truth value of compound sentences. Instead of truth tables, LEAN automatically completes this interpretation. Please do not change the first 2 lines of the code editor, they define the interpret function in LEAN. If you click on the #eval statements, you evaluate the interpret function on the compound sentences. In the lower half of the code editor under "Messages" you can see the value of the compound expression, 'true' or 'false'. Verify LEAN's evaluation for the given expressions using a truth table.

You have to be precise with your brackets in LEAN. First, because we evaluate the sentences using the interpret function, we need to put brackets around the sentences. If you use multiple connectives and don't specify the brackets, LEAN automatically adds brackets from left to right. The first example shows this: if you input $P \land \neg P \lor R$, LEAN will evaluate $(P \land \neg P) \lor R$.

Countermodels

We just looked at a toy model in the previous example, with given truth values for $P,Q,R$. Next, let's consider countermodels. As a reminder, a sentence $\phi$ is logically implied by a set of sentences $\Gamma$ (formally $\Gamma \vDash \phi$) if there is no counterexample to the argument. The next exercise is for you to define the values of P, Q and R such that the statements in the code editor are false. The task is to provide a countermodel.
