Semantics of Propositional Logic

This section will discuss the semantics of Propositional Logic, following Chapter 5 of the Logic Manual. Please do not consider the first 2 lines of the code editor, this defines the interpret function in LEAN.

Semantic structures assign truth values to sentences in the language. We construct here p toy model and declare sentences P, Q as false and R as true. Now following the definition of satisfaction (Def 5.2) you can evaluate the truth value of compound sentences. Instead of truth tables, LEAN automatically completes this interpretation. If you click on the #eval statements, you can see 'true' or 'false'.

You have to be precise with your brackets in LEAN. First, because we evaluate the interpret function, we need to put brackets around the sentences. If you use multiple connectives and don't specify the brackets, LEAN automatically adds brackets from left to right. The first example shows this: if you input $P \land \neg P \lor R$, LEAN will evaluate $(P \land \neg P) \lor R$.

Countermodels

We just looked at an example model in the previous example. Now we say that a sentence $\phi$ is logically implied by a set of sentences $\Gamma$ (formally $\Gamma \vDash \phi$) if there is no counterexample to the argument. The next exercise is for you to define the values of P, Q and R such that the statements in the code editor are false. So you provide a countermodel.
